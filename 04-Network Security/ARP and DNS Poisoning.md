# ARP poisoning (ARP spoofing)

- Как работает ARP?
1. Устройство спрашивает: Кто владелец IP 192.168.1.1? Сообщи свой MAC-адрес
2. Владелец отвечает: Это я, мой MAC — aa:bb:cc:dd:ee:ff

- Как работает ARP Poisoning 
1. Появляется злоумышленник: 192.168.1.14 
2. Он отправляет жертве сообщение: Я — 192.168.1.1 Мой MAC: 11:22:33:44:55:66
3. Жертва обновляет ARP Cache: 192.168.1.1 → MAC злоумышленника
4. Теперь весь трафик к роутеру сначала идёт злоумышленнику

# DNS Spoofing (DNS Poisoning)

- Как работает обычный DNS?
1. Пользователь запрашивает: professormesser.com
2. DNS отвечает: 203.0.113.10

- Как работает DNS Poisoning?
1. 1. Атакующий подменяет DNS-ответ в кеше резолвера
2. На запрос professormesser.com сервер возвращает IP фишингового сайта
3. Пользователь вводит professormesser.com, а попадает на поддельный сайт
4. Кража логинов, паролей, банковских данных

---

# Для собеседования 
- ARP poisoning — подмена MAC-адреса, чтобы трафик шёл через атакующего
- DNS poisoning — подмена DNS-ответа, чтобы перенаправить на фишинговый сайт
