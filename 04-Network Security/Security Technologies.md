# Honeypot / Honeynets

Honeypot — ложная цель (специально уязвимый сервер) для приманки злоумышленника.

Honeynets — сеть из нескольких honeypot'ов.

Зачем SOC:
- Узнать методы атакующего
- Отвлечь его от реальных целей
- Собрать информацию для улучшения защиты

Пример: Вы поднимаете сервер с открытым портом 22 (SSH) и слабым паролем `admin:12345`. Как только кто-то его ломает — всё его действия записываются в лог.

# CIA Triad (святая троица ИБ)

| Принцип | Что значит | Пример нарушения | Пример защиты |
|:--------|:-----------|:-----------------|:--------------|
| Confidentiality | Доступ только у авторизованных | Утечка базы данных клиентов | Шифрование, пароли |
| Integrity | Данные не изменены кем-то без прав | Подмена суммы платежа в банке | Хеширование, цифровые подписи |
| Availability | Система работает, когда нужна | DoS-атака на сайт | Резервирование, UPS, кластеризация |

---

# Для собеседования (одной строкой)
- CIA Triad — это три главных принципа безопасности: Confidentiality (конфиденциальность), Integrity (целостность), Availability (доступность).
- Confidentiality — доступ к данным только у тех, у кого есть права. Нарушение: утечка базы данных.
- Integrity — данные не были изменены без разрешения. Нарушение: подмена суммы платежа в банке.
- Availability — система работает, когда нужна. Нарушение: DoS-атака, отключение сервера.
- Honeypot — ловушка для хакера (специально уязвимый сервер), чтобы изучить его методы.
- Honeynet — сеть из нескольких honeypot'ов.
