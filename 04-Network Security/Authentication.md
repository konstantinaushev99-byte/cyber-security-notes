# Authentication

---

# AAA Framework (процесс)

| Этап | Что значит | Пример |
|:----:|:-----------|:-------|
| Identification | Вы называете себя | Вводите имя пользователя |
| Authentication | Вы доказываете, что это вы | Вводите пароль |
| Authorization | Система решает, что вам можно | Доступ к файлам по роли |

  Порядок важен: Identification → Authentication → Authorization

---

# Технологии аутентификации

| Технология | Что делает | Где используется |
|:-----------|:-----------|:------------------|
| Single sign-on(SSO) | Один пароль для всех сервисов | Корпоративный портал, Google Workspace |
| RADIUS(Remote Authentication Dial-In User Service) | Центральная аутентификация, авторизация, учёт | Wi-Fi Enterprise, VPN, коммутаторы |
| LDAP(Lightweight Directory Access Protocol) | Доступ к каталогу пользователей | Active Directory, OpenLDAP |
| SAML(Security Assertion Markup Language) | Обмен данными об аутентификации между системами | SSO в браузере (например, вход в Salesforce через Google) |

---

#  Факторы аутентификации (3 фактора)

| Тип | Что значит | Пример |
|:----|:-----------|:-------|
| Что ты знаешь (Knowledge) | То, что человек помнит | Пароль, PIN-код, ответ на секретный вопрос |
| Что ты имеешь (Possession) | То, что у человека есть | Токен, смарт-карта, телефон (SMS) |
| Кто ты есть (Inherence) | То, чем человек является | Отпечаток пальца, Face ID, голос |

MFA (Multi-Factor Authentication) — используется два или более разных факторов (например, пароль + код из SMS).

2FA — частный случай MFA (ровно два фактора).

---

 # SAML vs LDAP vs RADIUS

| Протокол | Для чего | Особенность |
|:---------|:---------|:------------|
| RADIUS | Сетевая аутентификация (Wi-Fi, VPN) | Работает по UDP, объединяет аутентификацию и учёт |
| LDAP | Доступ к каталогу (поиск пользователей) | Не занимается авторизацией сам, только хранит данные |
| SAML | SSO между веб-приложениями | Передаёт XML-сообщения о том, что пользователь уже залогинен |

---

# Для собеседования 

- AAA framework — это процесс: сначала Identification (кто ты), потом Authentication (докажи), потом Authorization (что тебе можно).
- Факторы аутентификации — три типа доказательств: что ты знаешь (пароль), что ты имеешь (токен), кто ты есть (отпечаток).
- MFA (Multi-Factor Authentication) — использование двух и более разных факторов (например, пароль + код из SMS).
- RADIUS — центральная аутентификация для сетевых устройств (Wi-Fi Enterprise, VPN, коммутаторы).
- LDAP — протокол доступа к каталогу пользователей (Active Directory).
- SAML — протокол для SSO между веб-приложениями (через браузер).
- SSO (Single Sign-On) — один пароль для доступа ко всем сервисам компании.
