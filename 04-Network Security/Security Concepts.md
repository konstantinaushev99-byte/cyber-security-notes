# PKI и управление доступом (Public Key Infrastructure)

Что это - Инфраструктура для управления цифровыми сертификатами и ключами шифрования

# Компоненты PKI:

### Digital certificates(цифровой сертификат)

Что это - электронное удостоверение, которое подтверждает: Этот сайт / устройство — действительно тот, за кого себя выдаёт

Где используется: HTTPS (TLS/SSL), подпись кода, email (S/MIME)

Содержимое сертификата:
- Публичный ключ
- Владелец (Common Name — домен)
- Кем выдан (CA)
- Срок действия
- Цифровая подпись CA

---

| Компонент | Что делает |
|:----------|:-----------|
| CA (Certificate Authority) | Выпускает и подписывает сертификаты |
| IAM (Identity and Access Management) | Управляет разрешениями и доступом |
| RBAC (Role-based access control) | Управление доступом на основе ролей |
| GR (Geographic Restrictions) | Выдача доступа в зависимости от местоположения |

 GR  - например, доступ к системе только из офиса или только из России

 ---

 # Состояния данных

| Состояние | Что значит | Пример защиты |
|:----------|:-----------|:--------------|
| Data at rest | Данные хранятся на диске | Шифрование диска (BitLocker, LUKS) |
| Data in transit | Данные передаются по сети | TLS/SSL, VPN, SSH |
| Data in use | Данные обрабатываются в памяти | Защита от дампов памяти |

---

# Для собеседования

- CA — кто выпускает сертификаты (HTTPS)
- IAM — кто управляет доступом
- RBAC — доступ по ролям (бухгалтер видит финансы, программист — код)
- GR — доступ по географии
