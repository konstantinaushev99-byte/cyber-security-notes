# ARP(Address Resolution Protocol)

---

# Work ARP
```bash
IP → MAC
```
Например:
Ты знаешь IP телефона: 192.168.3.94
Но для отправки кадра по Ethernet нужен MAC: E2:4A:C3E3:41:CC
ARP как раз спрашивает: Кто имеет IP 192.168.3.94?
И получает ответ: Это я, мой MAC E2:4A:C3:E3:41:CC

### Как выглядит ARP
- Запрос:
  ```bash
  Who has 192.168.3.94?
  Tell 192.168.3.109
- Перевод:
  ```bash
  Кто владеет IP 192.168.3.94?
  Сообщите 192.168.3.109
- Ответ:
  ```bash
  192.168.3.94 is at E2:4A:C3:E3:41:CC

# Итог:
- ARP отвечает на вопрос: "Я знаю IP. Какой у него MAC?"
- Сначало идет ARP, только потом ICMP(ping)
- Почему ARP Request — Broadcast?
  - Потому что отправитель не знает, какое устройство владеет нужным IP, поэтому спрашивает всю локальную сеть
- Почему ARP Reply — Unicast?
  - Потому что телефон уже знает, кто его спрашивал: Tell 192.168.3.109 , и отвечает только ему
