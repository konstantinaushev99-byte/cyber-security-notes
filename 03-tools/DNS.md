# Что такое DNS?
DNS(Domain Name System) - Преобразует доменые имена(google.com) , в IP адреса (142.250.185.46)

---

# Forward и Reverse Lookups 

| Тип | Что делает | Пример |
|:----|:-----------|:-------|
| Forward | Имя -> IP | google.com -> 142.250.185.46 | 
| Reverse | IP -> Имя | 8.8.8.8 → dns.google | 

Reverse lookup использует специальную зону in-addr.arpa (для IPv4)

---

# Типы DNS-серверов

| Тип | Что делает | Пример |
|:----|:-----------|:-------|
| Рекурсивный резолвер | Принимает запрос от клиента, сам находит ответ (обходит все серверы) | Ваш роутер, 8.8.8.8 |
| Авторитетный сервер | Хранит реальные записи для домена | Серверы Google для google.com |
| Корневой сервер | Указывает, где искать TLD (.com, .org) | 13 корневых серверов в мире |

Как работает запрос:
1. Клиент → Рекурсивный резолвер
2. Рекурсивный → Корневой сервер (→ TLD-сервер → Авторитетный)
3. Авторитетный → возвращает IP
4. Рекурсивный → возвращает клиенту и кеширует

---

# Атаки на DNS 

- DNS Poisoning (Cache Poisoning)

Что это: Подмена DNS-ответа в кеше резолвера

Как работает:
1. Злоумышленник отправляет поддельный DNS-ответ
2. Рекурсивный резолвер сохраняет его в кеш
3. Пользователи перенаправляются на фишинговый сайт

Пример: Вместо bank.com → реальный IP, жертва получает IP поддельного сайта

- DNS Tunneling

Что это: Кодирование данных в DNS-запросах (способ обходить файрволы)

Как обнаружить: Очень длинные имена (десятки случайных символов), большой объём DNS-трафика

- Защита
DNSSEC — подписывает DNS-ответы цифровой подписью (проверяет, что ответ подлинный)

---

# Типы DNS-записей (таблица для запоминания)

| Тип | Назначение | Пример |
|:----|:-----------|:-------|
| A    | IPv4-адрес домена | google.com -> 142.250.185.46 |
| AAAA | IPv6-адрес домена | google.com → 2a00:1450:401b:800::200e | 
| CNAME | Alias (одно имя указывает на другое | www.example.com → example.com | 
| MX    | Почтовый сервер для доменов | example.com → mail.example.com | 
| TXT   | Текстовая информация (SPF, DKIM для проверки писем) | v=spf1 include:_spf.google.com ~all |
| NS    | Авторитетный DNS-сервер для домена | ns1.google.com |
| PTR   | Reverse lookup (IP → имя) | 8.8.8.8 → dns.google |

---

# Для собеседования

DNS переводит имена в IP (A-запись) и наоборот (PTR). Рекурсивный резолвер ищет ответ, авторитетный — хранит. DNS poisoning — подмена ответа. DNSSEC — защита от подмены

# Что я проверил сам

```bash
# В терминале Debian

# 1. Forward lookup: узнать IP google.com
dig google.com A
# Вывод: 6 IPv4-адресов (142.251.1.102 и другие)
# SERVER: 192.168.3.1#53 — мой DNS-сервер (роутер)

# 2. Reverse lookup: узнать имя по IP 8.8.8.8
dig -x 8.8.8.8
# Вывод: 8.8.8.8 → dns.google

# 3. Узнать, какой DNS-сервер использует система
cat /etc/resolv.conf
# Вывод: nameserver 192.168.3.1

# 4. Альтернативная команда (проще для быстрой проверки)
nslookup google.com
# Вывод: Non-authoritative answer (пришло из кеша роутера)
