### Network Monitoring

# NetFlow

Что это - это сводная статистика, основаная на потоках трафика, проходящих через сеть.

**Что записывает:**
- Кто отправил (IP, порт)
- Кто получил (IP, порт)
- Сколько данных передано
- Время начала и конца сессии

Для SOC: Понимать, что в сети происходит, даже если трафик зашифрован.

# Syslog

Что это: Протокол для отправки логов с устройств на сервер (обычно на порт 514/UDP).

Уровни серьезности Syslog 
| Уровень | Название | Что значит |
|:-------:|:---------|:-----------|
| 0 | Emergency | Система не работает |
| 1 | Alert | Нужно срочно вмешаться |
| 2 | Critical | Критическая ошибка |
| 3 | Error | Ошибка |
| 4 | Warning | Предупреждение |
| 5 | Notice | Важное уведомление |
| 6 | Info | Информация |
| 7 | Debug | Отладка (много деталей) |

# SIEM(Security information and event management)

Что это - централизованная система сбора, хранения и анализа логов со всех устройств сети.

Что собирает SIEM:
- Syslog (логи от роутеров, свитчей, фаерволов, Linux)
- Windows Event Logs (логи от Windows-серверов)
- NetFlow (метаданные о трафике: кто с кем общался)
- Алерты от IDS/IPS

Зачем нужен SOC:
- Корреляция событий (связать логи с разных устройств)
- Поиск инцидентов
- Хранение логов для расследования

---

# API Integration (интеграция через API)

Что это: Способ подключения SIEM или других систем безопасности к API-шлюзам, облачным сервисам, приложениям для сбора логов и событий в реальном времени.

Как работает:
1. SIEM подключается к API сервиса (например, AWS CloudTrail, Salesforce, Okta)
2. Сервис отправляет логи/события через API в SIEM
3. SIEM анализирует данные и ищет аномалии
  
Зачем SOC:
- Современные приложения живут в облаке и не генерируют Syslog
- API Integration позволяет собирать логи оттуда, куда не поставить агент
- Обнаружение атак на API (например, брутфорс OAuth-токенов)

Примеры:
- SIEM получает логи от AWS CloudTrail (все API-вызовы в вашем облаке)
- SIEM подключается к API Slack/Google Workspace для сбора логов о пользователях

Для собеседования:
API Integration — это когда SIEM забирает логи не по Syslog, а через API облачного сервиса. Нужно для современных сред, где нет классических устройств.

# Port Mirroring (SPAN / зеркалирование порта)

Что это: Технология на коммутаторе, которая копирует трафик с одного порта (или VLAN) на другой порт, куда подключен анализатор (Wireshark, IDS/IPS).

Как работает:
1. На коммутаторе настраивается source port (откуда копируем) и destination port (куда отправляем копию)
2. Коммутатор дублирует все пакеты с source port на destination port
3. Анализатор (например, Wireshark или IDS) видит точную копию трафика

Зачем SOC:
- Анализ содержимого пакетов (в отличие от NetFlow, который даёт только метаданные)
- Обнаружение атак, которые не видны по логам (например, эксплойты в HTTP-трафике)
- Отладка без остановки производства

---

Чем отличается от NetFlow:

| Характеристика | Port Mirroring | NetFlow |
|:---------------|:---------------|:--------|
| Что даёт | Полные пакеты (payload) | Только метаданные (IP, порты, объём) |
| Объём данных | Огромный | Маленький |
| Для чего | Глубокий анализ, IDS/IPS | Мониторинг, учёт трафика |
| Может ли зашифрованный трафик? | Да, но payload не прочитать | Да, метаданные видны |

Для собеседования:
Port Mirroring (SPAN) копирует трафик с одного порта коммутатора на другой. Нужно, чтобы подключить IDS или Wireshark без разрыва соединения. В отличие от NetFlow, видит полные пакеты, а не только заголовки.
