# Users and Rights

1. whoami - текущий пользователь
  - Показывает имя пользователя, под которым вы вошли в систему
  - Пример:
  ```bash
  $ whoami
    kosta
  ```
2. id - подробная информация о пользователе
   - Показывает UID (идентификатор пользователя)
   - GID (идентификатор основной группы)
   - все группы, в которые входит пользователь
   - Синтаксис: id [имя_пользователя]
   - Пример:
     ```bash
      $ id
      uid=1000(kosta) gid=1000(kosta) groups=1000(kosta),24(cdrom),27(sudo),100(users)

     $ id root
     uid=0(root) gid=0(root) groups=0(root)

3. sudo - выполнение команд от имени root
   - Выполняет команду с правами другого пользователя (обычно root)
   - Пример:
     ```bash
     $ whoami
     kosta

     $ sudo whoami
     root

4. chmod - изменение прав доступа (ключевая тема)
   - Меняет права доступа к файлам и папкам
   - Синтаксис: chmod [права] <файл/папка>

     ---

    Три типа доступа
   | Буква | Значение | Для файла | Для папки |
   |:-----:|:---------|:----------|:----------|
   | r (read) | Чтение | Просмотр содержимого | Просмотр списка файлов (`ls`) |
   | w (write) | Запись | Изменение содержимого | Создание/удаление файлов |
   | x (execute) | Выполнение | Запуск как программы | Вход в папку (`cd`) |

   ---

    Три категории пользователей
   | Категория | Обозначение | Кто |
   |:----------|:-----------:|:----|
   | u (user) | Владелец | Тот, кто создал файл |
   | g (group) | Группа | Пользователи в группе владельца |
   | o (others) | Остальные | Все остальные |
   | a (all) | Все | u + g + o вместе |

   ---

    Два способа задания прав

    Способ 1. Символьный (буквы)
    | Команда | Результат |
    |:--------|:----------|
    | chmod u+x script.sh | Добавить владельцу право на выполнение |
    | chmod g-w file.txt | Убрать у группы право на запись |
    | chmod o=r README.md | Установить остальным только чтение |
    | chmod a+rwx file | Дать все права всем |
    | chmod -R u+rwx folder/ | Рекурсивно (для папки и всего внутри) |

    Пример:
    ```bash
    $ ls -l script.sh
    -rw-r--r-- 1 kosta kosta 0 Jun 10 10:00 script.sh

    $ chmod u+x script.sh

    $ ls -l script.sh
    -rwxr--r-- 1 kosta kosta 0 Jun 10 10:00 script.sh
    ```
    
    Способ 2. Числовой(восьмеричный)
    | Цифра | Права | Символьный вид | 
    |:------|:------|:---------------|
    |   0   | нет прав | --- |
    |   1   | Только выполнение | --x |
    |   2   | Только запись | -w- |
    |   3   | Запись + выполнение (2+1) | -wx |
    |   4   | Только чтение | r-- |
    |   5   | Чтение + выполнение (4+1) | r-x |
    |   6   | Чтение + запись (4+2) | rw- |
    |   7   | Всё (4+2+1) | rwx |

5. chown - смена владельца
   - Меняет владельца и/или группу файла/папки
   - Синтаксис: chown [новый_владелец][:новая_группа] <файл>
   - ### Примеры
    | Команда | Результат |
    |:--------|:----------|
    | sudo chown alice file.txt | Владельцем становится alice |
    | sudo chown :developers file.txt | Меняет группу* на developers |
    | sudo chown alice:developers file.txt | Меняет и владельца, и группу |
    | sudo chown -R alice:alice folder/ | Рекурсивно (для всей папки и всего внутри) |

     Пример:
     ```bash
     $ ls -l file.txt
     -rw-r--r-- 1 root root 0 Jun 10 10:00 file.txt

     $ sudo chown kosta:users file.txt

     $ ls -l file.txt
     -rw-r--r-- 1 kosta users 0 Jun 10 10:00 file.txt
