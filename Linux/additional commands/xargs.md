# xargs 
### Что делает - берёт вывод одной команды и подставляет его как аргументы для другой команды. Это принципиально другой механизм — не обработка текста построчно, а построение и запуск новой команды
### Зачем нужен - Многие команды (например, nmap, rm, cp, ping) не читают данные из пайпа (|) напрямую — им нужны аргументы

---

# Примеры 
1. с построчным вводом
```bash
echo -e "192.168.1.1\n192.168.1.2\n192.168.1.3" | xargs -n1 ping -c1

-n1 — говорит xargs подставлять по одному аргументу за раз, то есть запустить ping -c1 отдельно для каждого IP, а не сразу для всех вместе
```
2. кейс с nmap
```bash
В прошлом уроке я получил список открытых портов:
cat ports.txt | awk '/open/ {print $1}' | cut -d'/' -f1

22
443

Теперь представим, что вы хотите просканировать именно эти порты отдельной, более глубокой командой (-sV)
cat ports.txt | awk '/open/ {print $1}' | cut -d'/' -f1 | xargs -I{} nmap -sV -p{} 192.168.3.127

Разберём -I{}:
-I задаёт placeholder (заполнитель) — здесь это {}
Каждое значение, пришедшее по пайпу, подставляется вместо {} в команде
То есть для порта 22 выполнится nmap -sV -p22 192.168.1.1, для 443 — nmap -sV -p443 192.168.3.127
```

# Ключевые флаги xargs
| Флаг | Что делает |
|------|------------|
| -n1  | Подставлять по одному аргументу за раз (запускать команду отдельно для каждого) |
| -I{} | Задать placeholder — куда именно вставлять значение (нужно, если значение не в конце команды, а посередине) |
| -p   | Спросить подтверждение перед каждым запуском (полезно для опасных команд типа rm) |
| -P4  | Запускать до 4 команд параллельно (ускоряет массовые операции) |
| -a файл | Взять аргументы из файла напрямую, без cat |

# Разница с awk/cut — важно понять
- cut и awk — обрабатывают текст, их вывод остаётся текстом
- xargs — строит и запускает новую команду, используя полученный текст как аргументы
