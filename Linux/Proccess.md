#  Управление процессами

---

1. ps — процессы в текущей сессии
   Описание: Показывает процессы, запущенные в текущем терминале.
   ```bash
   ps

   PID TTY          TIME CMD
   1234 pts/0    00:00:01 bash
   5678 pts/0    00:00:00 ps

---

2. ps aux - Показывает все процессы с подробной информацией
   ```bash
   ps aux

   USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
   root         1  0.0  0.1  16836  1128 ?        Ss  10:00   0:01 /sbin/init
   kosta     1234  0.0  0.2  11524  2048 pts/0    Ss  10:01   0:00 bash
   kosta     5678  0.0  0.1  15764  1048 pts/0    R+  10:05   0:00 ps aux

  Синтаксис: ps aux
  Опция	Значение
  - a	Все процессы (не только текущего пользователя)
  - u	Пользовательский формат (с полями USER, %CPU, %MEM)
  - x	Процессы, не привязанные к терминалу (фоновые/демоны)

  Что означают поля:
  - Поле	Значение
  - USER	Владелец процесса
  - PID	ID процесса
  - %CPU	Использование CPU в процентах
  - %MEM	Использование памяти
  - VSZ	Виртуальная память (КБ)
  - RSS	Физическая память (КБ)
  - TTY	Терминал (? — фоновый процесс)
  - STAT	Состояние процесса
  - START	Время запуска
  - TIME	Общее время CPU
  - COMMAND	Команда

  Состояния процесса (STAT):
  Код	Значение
  - R	Running (выполняется)
  - S	Sleeping (ждёт)
  - D	Uninterruptible sleep (ожидание диска)
  - Z	Zombie (зомби — процесс завершён, но не убран родителем)
  - T	Stopped (остановлен)
  - "+" Процесс на переднем плане (foreground)

---

3. top - Динамический просмотр процессов, обновляется каждые 2–3 секунды
   Интерактивные клавиши внутри top:
   - q	Выйти
   - h	Справка
   - k	Убить процесс (запросит PID и сигнал)
   - r	Изменить приоритет (renice)
   - P	Сортировка по %CPU
   - M	Сортировка по %MEM
   - 1	Показать все CPU (если несколько ядер)

     ```bash
     top

     top - 10:30:01 up 1 day,  2:15,  2 users,  load average: 0.05, 0.03, 0.01
     Tasks: 123 total,   1 running, 122 sleeping,   0 stopped,   0 zombie
     %Cpu(s):  2.5 us,  1.2 sy,  0.0 ni, 96.0 id,  0.3 wa,  0.0 hi,  0.0 si,  0.0 st
     MiB Mem :   7958.4 total,   3456.2 free,   1234.5 used,   3267.7 buff/cache
     MiB Swap:   2048.0 total,   2048.0 free,      0.0 used.   6234.5 avail Mem

     PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND
     1234 kosta     20   0  11524   2048   1024 S   0.3   0.1   0:01.23 bash
     5678 root      20   0  16836   1128    896 S   0.0   0.1   0:00.56 systemd

---

4. htop - Альтернатива top с цветным интерфейсом, поддержкой мыши и удобным управлением
   Интерактивные клавиши в htop:
   - F10 или q	Выйти
   - F9	Убить процесс
   - F6	Сортировка
   - F4	Поиск (фильтр)
   - F3	Инкрементальный поиск
   - F2	Настройки
   - Space	Отметить процесс

---

5. kill - Отправляет сигнал процессу (по умолчанию — SIGTERM для мягкого завершения)
   Основные сигналы:
   - SIGTERM	15	Мягкое завершение (по умолчанию). Процесс может сохранить данные.
   - SIGKILL	9	Жёсткое завершение. Процесс убивается мгновенно, без сохранения.
   - SIGINT	2	Прерывание (как Ctrl+C)
   - SIGHUP	1	Перезагрузка конфигурации (без остановки)

---

6. kill -9 - Отправляет сигнал SIGKILL, который немедленно завершает процесс без возможности сохранить данные
   Когда использовать:
   - Процесс завис и не реагирует на обычный kill
   - Процесс в состоянии D (ожидание диска) или Z (зомби) — часто не убивается даже -9
   Когда НЕ использовать:
   - Если процесс может потерять важные данные (например, база данных). Сначала пробуйте kill (без опций) или kill -15.
