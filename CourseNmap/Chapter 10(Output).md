# Форматы 
1. -oN(Normal)
   - Удобен для обычного просмотра(как в терминале)
   - Пример:
     ```bash
     22/tcp open ssh
     80/tcp open http
     443/tcp closed https
     ```
    - Когда использовать? - Когда ты сам хочешь открыть файл и почитать результаты
2. -oX(XML)
   - Удобен для программ
   - Пример:
     ```bash
     <port protocol="tcp" portid="22">
         <state state="open"/>
         <service name="ssh"/>
     </port>
     ```
   - Когда использовать? - Когда результат будет обрабатывать Python, Bash, PowerShell, веб-приложение или другие программы
3. -oG(Grepable) - устаревший способ, но его все еще можно встретить в старых скриптах
   - Удобен для поиска и сканирования
   - Пример:
     ```bash
     Host: 192.168.3.116 Ports: 22/open/tcp//ssh///
     Такой формат очень удобно искать через:
     grep "22/open"
     ```
4. -oA
   - Сохраняет сразу 3 формата
   - Пример:
     ```bash
     nmap -sS -oA scan 192.168.3.116
     Создаст сразу 3 пакета:
     scan.nmap
     scan.xml
     scan.gnmap

# Практика
1. Выполню 4 команды в терминале:
   ```bash
   nmap -sS -oN scan.nmap 192.168.3.116
   nmap -sS -oX scan.nmap 192.168.3.116
   nmap -sS -oG scan.nmap 192.168.3.116
   nmap -sS -oA fullscan 192.168.3.116
2. После проверю какие файлы появились с помощью команды - ls
3. В итоге я увидел 3 файла  Fullscan в форматах gnmap,nmap,xml, также я увидел 1 файл scan.nmap
4. Самый удобный среди них остается - fullscan.nmap - потому что выглядит также как вывод в терминале
