# Decoys - второй метод Firewall
Что это - техника, которая не скрывает настоящий IP, а создаёт несколько ложных IP, 
чтобы усложнить определение реального источника сканирования

---

# Как работает?
1. Настоящий IP:
   ```bash
   → SYN
   ← SYN/ACK
   → ACK
   → следующий запрос
2. Decoy IP:
   ```bash
   → SYN
   ← SYN/ACK
   (дальше тишина)


---

# Как администратор находит настоящий IP?
Он смотрит не только на первый пакет SYN, а на дальнейшее поведение.

Настоящий атакующий:
- устанавливает TCP-соединение
- отвечает на SYN/ACK
- отправляет новые запросы
Decoy:
- отправляет только SYN
- не продолжает соединение

Поэтому настоящий IP можно определить по активному продолжению общения.

---

# Минус Decoy
Decoy создаёт только путаницу среди источников.
Он не делает атакующего анонимным, потому что настоящий источник 
отличается своим поведением
