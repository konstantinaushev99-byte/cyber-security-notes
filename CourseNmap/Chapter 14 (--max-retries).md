# --max-retries
Что это - задает максимальное количество повторных отправок пакета при отсутствии ответа

---

# Пример 1
Если Nmap не получает ожидаемый ответ, он может повторить отправку пакета, чтобы проверить, был ли пакет потерян или ответ заблокирован
```bash
sudo nmap --max-retries 5 192.168.3.116
SYN -> 192.168.3.116
Нет ответа
Nmap повторяет до 5 раз
Нет ответа после всех попыток
Nmap делает вывод:
порт вероятно filtered
```
---

# Пример 2
sudo nmap --max-retries 0 192.168.3.116
SYN -> цель
Нет ответа
Повторов нет
Nmap не делает повторных попыток и сразу переходит к следующему этапу сканирования (или делает вывод на основе первой попытки)
