# --spoof-mac
Что делает - Подменяет MAC-адрес отправителя
Работает только в локальной сети (L2). Через маршрутизаторы MAC переписывается
Для чего?
- попытаться обойти фильтрацию по MAC
- выдать себя за другое устройство
- протестировать системы, использующие MAC-фильтрацию

---

# Пример
- sudo nmap --spoof-mac Cisco 192.168.3.116
- sudo nmap --spoof-mac 00:11:22:33:44:55 192.168.3.116
