# Firewall Evasion
 Что такое Firewall - это система безопасности, которая контролирует и фильтрует сетевой трафик между устройством (компьютером, сервером, локальной сетью) и внешней сетью 

 ---

# Первый метод Fragmentation
Суть метода - раньше некоторые firewall не собирали IP-фрагменты перед анализом, поэтому техники фрагментации иногда работали. Современные firewall обычно сначала собирают фрагменты обратно в один пакет, а затем анализируют его 

Как работает:
1. Злоумышленник разбивает TCP-сегмент или ICMP-запрос на мелкие куски
2. Первый фрагмент содержит начало TCP-заголовка (с портом 80 или 443), поэтому файрвол пропускает его
3. Последующие фрагменты не содержат информации о порте, если firewall не собирает фрагменты перед анализом, ему может не хватить информации для принятия правильного решения
4.На целевой машине IP-стек собирает все фрагменты обратно в один пакет и передает его TCP

# Практика
1. Сперва в Wireshark  поставил фильтр - ip.addr == 192.168.3.116 (Ubuntu)
2. После настройки Firewall на другой VM, я решил его проверить командой:
   ```bash
   sudo nmap -sS -p 22,80,443 <IP_новой_Ubuntu>
   Команду выполнил на основной VM
3. После выполнил сканирование с помощью фрагментации:
   ```bash
   sudo nmap -sS -f -p 22 192.168.3.116
4. После это вернулся в Wireshark и появились такие строки, которые доказывают что фрагментация была выполнена <img width="1258" height="37" alt="image" src="https://github.com/user-attachments/assets/c07284b0-1c8a-4973-bc46-a80804f29e24" />
5. несмотря на использование -f, результат сканирования не изменился (22/tcp open). Это означает, что современный firewall (или сетевой стек) корректно обработал фрагментированные пакеты
