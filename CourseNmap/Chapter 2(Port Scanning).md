# Port Scanning - сканирование портов

---

# Что такое порт - порт это логическая точка входа в сетевую службу
Например:
  ```bash
  IP

  192.168.3.14

        │
        │
        ├── Port 22 → SSH
        ├── Port 80 → Web Server
        ├── Port 443 → HTTPS
        └── Port 3306 → MySQL
```

---

# Что делает Nmap?
Например:
```bash
nmap 192.168.3.14
И он начинает сканировать порты
22?
↓
закрыто
80?
↓
открыто
443?
↓
открыто
445?
↓
закрыто

После этого выводин:
22 closed
80 open
443 open
```
И делает это все он с помощью TCP, и да, тут можно вспомнить как работает TCP Handshake

Обычный TCP Handshake делает это:
Клиент(SYN) --> Сервер(SYN/ACK) --> Клиент(ACK)

Но Nmap поступает по другому, и вот что он делает при использование nmap -sS:
Nmap(SYN) --> Сервер(SYN/ACK) --> Nmap(RST)
Последнего ACK нету, вместо него RST, потому что цель nmap не подключиться к порту, а узнать открыт он, или нет

Но если порт закрыт вот что происходит:
Nmap(SYN) --> <-- Сервер(RST) сервер сам говорит что этот порт закрыт
