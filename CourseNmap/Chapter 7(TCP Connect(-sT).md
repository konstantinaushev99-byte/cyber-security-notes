# Как это раюотает?
1. Во 2 главе мы проходили сканирование портов с помощью -sS, теперь мы будет проверять это с помощью -sT
2. Вспомним как работает обычный TCP handshake: SYN -> SYN,ACK -> RST, после отправки пакетов сервером, nmap разрывает соединение и понимает что порт открыт, если сервер сам разрывает соединение, то порт закрыт
3. Теперь как работает -sT, начнем с того что для -sT не нужны root права как для -sS, также -sT делает полное подключение в отличие от -sS
4. И его подключение выглядит так: SYN -> SYN,ACK -> ACK , после полного подключения происходит следующее: FIN,ACK -> ACK -> FIN,ACK -> ACK
5. Тоесть он просто минует момент с паролем и после подключние, nmap понимает что порт открыт и подключится к нему возможно

| Вопрос | -sS | -sT |
|--------|-----|-----|
| Выполняется полный TCP Handshake? | Нет | Да |
| Требуется root-права? | Да | Нет |
| Более заметен в логах | Нет | Да |

# Когда что можно использовать
-sS — SYN scan (полуоткрытое сканирование)
Используешь, когда:
1. У тебя есть root
2. Нужно быстро просканировать много машин
3. Когда хочешь меньше "шума"
-sT — TCP Connect scan
Используешь, когда:
1. Нет root-прав
2. Нужно проверить именно возможность реального подключения
3. В корпоративной среде, где raw packets запрещены

# Итог
Главное отличие в одной таблице
| Вопрос | -sS | -sT |
|--------|-----|-----|
| Требует root | Да | Нет |
| Работает через | Raw packets | ОС TCP stack |
| Handshake | Не полный | Полный | 
| Скорость | Быстрее | Медленнее | 
| Соединение появляется в логах сервиса | Обычно нет | Да | 
| Реалистичность подключения | Ниже | Выше | 
| Используется чаще | Пентесты/админы | Обычные пользователи |
